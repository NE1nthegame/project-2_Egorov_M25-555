## Новые возможности (Декораторы и замыкания)

### Обработка ошибок
Все операции базы данных теперь защищены декоратором `@handle_db_errors`, который перехватывает:
- `FileNotFoundError` - файлы данных не найдены
- `KeyError` - обращение к несуществующим таблицам или столбцам  
- `ValueError` - ошибки валидации данных
- Любые другие исключения

### Подтверждение опасных операций
Опасные операции запрашивают подтверждение:
- `drop_table` - удаление таблицы
- `delete` - удаление записей

### Замер производительности
Операции `insert` и `select` измеряют время выполнения с помощью декоратора `@log_time`.

### Кэширование запросов
Результаты запросов `select` кэшируются для одинаковых условий, что ускоряет повторные запросы.
